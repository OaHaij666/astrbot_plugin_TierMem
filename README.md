# TierMem-长期记忆 v2

TierMem-长期记忆是一个面向 AstrBot 的长期记忆插件。它可以记住用户的偏好、经历和近期事项，也能整理用户、群聊、项目与话题之间的关系，让 Bot 在后续对话中更自然地接上上下文。

插件不要求配置 Embedding 服务或向量数据库，安装后使用 SQLite 保存数据。

## 主要功能

- **多层记忆**：区分长期信息、一般事实、阶段性经历和短期事项。
- **自然衰减**：记忆会随时间减弱，再次提及或确认后会得到强化。
- **关系记忆**：记录用户与人物、群聊、项目、组织和兴趣之间的关系。
- **自动总结**：对话达到数量阈值，或等待超过设定时间后，自动整理为长期记忆。
- **群聊隔离**：用户私有记忆与群公共记忆按会话范围召回，避免跨群泄漏。
- **被动群聊观察**：可选地整理没有直接呼叫 Bot 的普通群消息，默认关闭。
- **可视化控制台**：查看记忆、关系图谱、召回结果和运行配置。

## 安装

推荐直接在 AstrBot 的插件市场中安装 TierMem。

也可以进入 AstrBot 插件目录后手动安装：

```bash
git clone https://github.com/OaHaij666/astrbot_plugin_TierMem.git
```

然后重启 AstrBot，或在插件管理页面重新加载插件。TierMem 需要 Python 3.10+，依赖会由 AstrBot 自动安装。

> TierMem v2 不迁移旧版数据库。检测到不兼容的旧数据结构时，会重新建立数据库。如需继续使用旧版，请参阅下方“使用旧版”。

## 快速开始

安装后，插件默认会：

1. 收集用户与 Bot 的近期对话。
2. 在对话达到阈值或等待超时后自动总结。
3. 在下一次相关对话中召回合适的记忆和关系。
4. 随时间降低长期未确认信息的影响。

通常无需修改配置即可开始使用。建议先确认 AstrBot 已配置可用的对话模型；如果需要单独指定总结模型，可在插件配置中填写 Provider ID。

## 默认记忆时长

TierMem v2 的默认值更适合互联网聊天：最稳定的信息也以 30 天为半衰期基准，短期信息会更快淡出。

| 记忆类型 | 适合保存的内容 | 默认半衰期 |
|---|---|---:|
| 核心记忆 | 身份、长期目标、稳定偏好 | 30 天 |
| 一般记忆 | 事实、知识与日常偏好 | 21 天 |
| 事件记忆 | 带时间背景的经历与事件 | 7 天 |
| 工作记忆 | 当前任务和短期事项 | 2 天 |
| 关系记忆 | 人物、项目与话题关系 | 30 天 |

“半衰期”表示记忆影响减半所需的时间，并不等于到期立即删除。重新提及、明确确认或强化会延长记忆的有效时间。以上数值可以在控制台中修改；更新插件默认值不会改写数据库中已有记忆的衰减速度。

## 被动群聊观察

该功能可以让 TierMem 了解群里正在讨论的公共事项，即使消息没有直接呼叫 Bot。

出于隐私和资源消耗考虑，它具有以下默认限制：

- 功能默认关闭。
- 开启后默认使用**白名单**，名单为空时不会观察任何群。
- 可以切换到黑名单模式，但黑名单为空会允许所有群，请谨慎使用。
- 每个群单独保存缓存和公共记忆，不会写入某个成员的私人记忆。
- Bot 自己的消息、命令、过短消息和重复消息会被跳过。

可在“TierMem 控制台 → 运行配置 → 被动群聊观察”中填写群 ID，并调整总结条数、最长等待时间和缓存上限。启用前请确保使用方式符合群成员预期及当地隐私规定。

## 可视化控制台

新版 AstrBot 会自动发现 TierMem 的管理页面。控制台可以用于：

- 查看记忆数量和运行状态。
- 搜索用户记忆并查看当前有效强度。
- 浏览实体与关系知识图谱。
- 测试某段文本会召回哪些记忆。
- 调整 FIFO、召回、衰减、群聊观察和总结模型配置。

保存配置后会立即应用到当前插件实例。

## 常用命令

| 命令 | 用途 |
|---|---|
| `/memory sum` | 立即总结当前近期对话 |
| `/memory check [layer]` | 查看当前用户的记忆 |
| `/memory graph` | 查看当前用户的关系 |
| `/memory fifo` | 查看尚未总结的近期对话 |
| `/memory status` | 查看记忆与关系统计 |
| `/memory clear` | 清除当前用户的数据 |
| `/memory rollback` | 管理员回滚数据库备份 |
| `/memory help` | 查看命令帮助 |

## 常见问题

### 为什么刚说过的话没有立刻成为长期记忆？

TierMem 会先把近期对话放入缓冲区，达到对话数量或最长等待时间后再总结。需要立即整理时可以使用 `/memory sum`。

### 为什么一条旧记忆没有被使用？

它可能与当前话题无关，也可能已经明显衰减。再次明确提及相同事实通常会强化它。

### 为什么开启被动群聊观察后没有记录？

默认使用白名单。请确认当前群 ID 已添加到名单，并检查总结条数和最长等待时间配置。

### 更新插件后加载异常怎么办？

先在 AstrBot 插件管理中停止并重新加载 TierMem。如果旧模块仍被占用，完整重启一次 AstrBot 即可清除 Python 模块缓存。
