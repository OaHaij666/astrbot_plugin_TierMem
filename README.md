# TierMem-长期记忆 v2

TierMem 是一款为 AstrBot 提供长期记忆能力的插件。

安装后，Bot 可以记住用户提到的个人信息、喜好、经历和近期计划，并在后续聊天中自然地回忆起来。TierMem 还会整理人物、群聊、项目和话题之间的关系，帮助 Bot 更好地理解聊天上下文。

无需配置 Embedding 模型或向量数据库。

## 功能介绍

- 记住用户的身份、喜好、经历和近期事项。
- 在后续聊天中自动召回与当前话题有关的记忆。
- 整理用户与人物、项目、组织、群聊和话题之间的关系。
- 自动总结近期对话，不需要手动维护记忆。
- 让长期没有再次提到的信息逐渐淡出，减少过时内容的影响。
- 分开保存私人记忆和群聊公共记忆，避免不同会话互相干扰。
- 提供可视化控制台，方便查看和管理记忆与关系。

## 安装

推荐在 AstrBot 的插件市场中搜索 `TierMem-长期记忆` 并安装。

也可以进入 AstrBot 的插件目录手动安装：

```bash
git clone https://github.com/OaHaij666/astrbot_plugin_TierMem.git
```

安装完成后，在 AstrBot 插件管理页面加载 TierMem。如果插件没有立即出现，重启一次 AstrBot。

## 开始使用

TierMem 安装后即可工作。请确保 AstrBot 已经配置了可用的聊天模型。

日常聊天会先进入近期对话记录。达到一定数量，或等待一段时间后，TierMem 会自动整理其中值得保留的内容。之后遇到相关话题，Bot 会自动参考这些记忆。

默认设置适合直接使用。如果需要调整，可以在插件配置或 TierMem 控制台中修改：

- 自动总结的对话数量和最长等待时间。
- 每次提供给 Bot 的记忆和关系数量。
- 不同类型记忆的保留速度。
- 私聊与群聊中的记忆使用方式。
- 用于整理记忆的模型。

## 记忆如何变化

TierMem 会根据内容类型保存不同记忆。例如，身份和稳定喜好会保留得更久，临时任务和短期计划则会更快淡出。

记忆淡出不代表立即删除，而是它对后续聊天的影响会逐渐降低。如果用户再次提到或确认同一件事，这条记忆会得到加强。

你可以在控制台的“运行配置 → 衰减半衰期”中调整各类记忆的保留速度。数值越大，记忆保持影响的时间越长；设置为 `0` 的项目不会随时间自动减弱。

## 被动群聊观察

开启后，TierMem 可以整理群内没有直接呼叫 Bot 的普通聊天，并形成只属于该群的公共记忆。例如群成员共同讨论的活动、项目进度或近期安排。

该功能默认关闭。启用时建议使用默认的白名单模式，并明确填写允许观察的群 ID：

- 白名单为空时，不会观察任何群。
- 黑名单模式下，空名单表示允许所有群。
- 群聊公共记忆不会写入某个成员的私人记忆。
- 一个群的公共记忆不会在私聊或其他群中使用。
- Bot 自己的消息、命令、过短消息和重复消息会被忽略。

可在“TierMem 控制台 → 运行配置 → 被动群聊观察”中启用并设置群名单。使用前请确认群成员知情并同意。

## 可视化控制台

在 AstrBot 的插件页面打开 TierMem 控制台，可以：

- 查看记忆、用户、实体和关系数量。
- 搜索和查看已经保存的记忆。
- 浏览用户与事物之间的关系图谱。
- 输入一段话，查看它会召回哪些记忆。
- 查看尚未总结的近期对话。
- 修改总结、召回、记忆衰减和群聊观察设置。

保存后，新配置会立即应用。

## 常用命令

| 命令 | 用途 |
|---|---|
| `/memory sum` | 立即整理当前近期对话 |
| `/memory check [layer]` | 查看当前用户的记忆 |
| `/memory graph` | 查看当前用户的关系 |
| `/memory fifo` | 查看尚未整理的近期对话 |
| `/memory status` | 查看记忆与关系统计 |
| `/memory clear` | 清除当前用户的记忆数据 |
| `/memory rollback` | 由管理员恢复数据库备份 |
| `/memory help` | 查看命令帮助 |

执行 `/memory clear` 会清除当前用户的记忆，请确认后再使用。

## 从旧版升级

TierMem v2 使用了新的记忆结构，不会导入旧版数据库。更新到 v2 后会建立新的数据库。

如果仍需使用旧版，可以安装 `v1-legacy` 分支：

```bash
git clone -b v1-legacy https://github.com/OaHaij666/astrbot_plugin_TierMem.git
```
